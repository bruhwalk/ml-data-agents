# Quality Report Generator

Generates a notebook with raw-vs-cleaned side-by-side comparison and justification markdown cell.

## Trigger

Called by the data-detective skill after the user picks a strategy and cleaned.parquet is saved.

## Autonomy rules

- **FULLY AUTONOMOUS**. No human interaction. Do everything silently.
- Generate notebook → execute → save with outputs. No questions.

## Workflow

### Step 1 — Prepare data

At this point you have:
- `data/raw/combined.parquet` — original dataset
- `data/cleaned/cleaned.parquet` — cleaned with chosen strategy
- `data/detective/problems.json` — detected issues
- `data/detective/QUALITY_REPORT.md` — comparative report (already generated by data-detective)

### Step 2 — Generate notebook

Create/update `notebooks/data_quality.ipynb` with cells **tailored to the actual data**. The notebook MUST include:

1. **Title + description** (markdown) — "Data Quality Report: [task description]"

2. **Load data** — read raw + cleaned

3. **All detected problems** (code + output):
   - Missing values — bar chart of nulls per column
   - Duplicates — count + examples
   - Text length distribution — histogram with IQR bounds, outliers highlighted
   - Class balance — bar chart of class counts (before label normalization)
   - Empty texts — count and percentage
   - Any additional problems the LLM found (near-duplicates, encoding issues, etc.)
   - Save each plot to `data/detective/` as PNG

4. **Label normalization** (markdown + code):
   - Show mapping applied
   - Class balance after normalization

5. **Raw vs Cleaned — side-by-side comparison** (code + output):
   - Comparison table as pandas DataFrame:
     | Metric | Raw | Cleaned | Change |
   - `data/detective/before_after_rows.png` — row count comparison
   - `data/detective/before_after_class_balance.png` — class distribution side by side
   - `data/detective/before_after_text_lengths.png` — overlapping histograms

6. **Justification** (markdown cell) — THIS IS CRITICAL:
   - What problems were detected and their severity
   - What was fixed and why it was necessary
   - What was kept and why it's acceptable
   - How the cleaning affects the downstream ML model (training quality, class balance, data size trade-off)
   - Must reference ACTUAL numbers from the data
   - Must be specific to THIS task, not generic

7. **Summary** — key findings and next steps

**Important:**
- Use `matplotlib` for plots (works headless)
- Save every plot to `data/detective/` as PNG
- Only compare raw vs cleaned (NOT all strategies — that was shown to user in Step 3 of data-detective)
- The justification must be SPECIFIC with real numbers, not generic boilerplate

### Step 3 — Execute the notebook

```bash
python .claude/skills/quality-report/scripts/run_notebook.py notebooks/data_quality.ipynb
```

### Step 4 — Verify

Check that these files exist and are non-empty:
- `notebooks/data_quality.ipynb` (with outputs)
- `data/detective/QUALITY_REPORT.md`
- `data/detective/before_after_class_balance.png`
- `data/detective/before_after_text_lengths.png`
- `data/detective/outliers.png`
- `data/detective/class_balance.png`

If any are missing, fix the notebook and re-run.
